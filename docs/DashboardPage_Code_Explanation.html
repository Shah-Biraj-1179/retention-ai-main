1 · Project Overview – What Does This File Do?

FoodRetainAI is an AI-powered web application that helps food-delivery businesses understand and prevent customer churn (customers who stop using the service).


DashboardPage.tsx is the heart / brain of the entire project. It is the page that opens after a user logs in. Everything useful — charts, predictions, insights, data tables, reports — lives inside this single file.

🎯 Simple one-line explanation for your guide:
"DashboardPage.tsx is the main analytics dashboard. It loads a dataset of customers, calculates statistics, shows charts and KPIs, lets users predict if a customer will churn, gives AI-based insights, and generates reports — all in one React component."

src/pages/DashboardPage.tsx
Navigated to via /dashboard route

🔗 What pages does it use?

Uses components from ChurnCharts, PredictionForm, and utility functions from dataset.ts

👤 Who sees it?

3 · State Variables – The "Memory" of the Dashboard

In React, state variables are like the RAM of a page — they hold information that can change while the user is interacting with the dashboard. When a state variable changes, React re-renders (re-draws) the relevant part of the page automatically.

Syntax: const [variableName, setVariableName] = useState(initialValue);
— variableName reads the current value  |  setVariableName(...) changes it
VariableTypeInitial ValuePlain English Purpose
data CustomerRecord[] [] Stores the full array of customer rows loaded from the CSV dataset. Each row has customer ID, city, age, orders, rating, delivery status, etc.
stats DatasetStats | null null Stores the computed statistics object (total customers, churn rate, average rating, churn by city, etc.) derived from data
loading boolean true Controls whether to show the loading spinner. Once the dataset is ready, it becomes false
importing boolean false Becomes true while a user-uploaded CSV is being parsed. Disables the import button to prevent double-clicks

dataFilter 'All' | 'Active' | 'Churned' 'All' Filters the data table to show All / only Active / only Churned customers

4 · Data Loading (useEffect) – How Data Gets Into the App
useEffect(() => { loadDataset().then((records) => { // Step 1: Load the CSV setData(records); // Step 2: Save raw rows setStats(computeStats(records)); // Step 3: Calculate statistics setLoading(false); // Step 4: Hide loading spinner }); }, []); // [] means: run only ONCE when the page first opens

📌 What is useEffect?

useEffect is a React hook that runs code after the component appears on screen. The empty array [] at the end means "run this only once when the page first loads" — not on every re-render.

📌 What does loadDataset() do?

It fetches the file public/data/dataset.csv using the browser's fetch() API, passes it through PapaParse to convert CSV text into JavaScript objects, then cleans/preprocesses the rows.

📌 The Loading State

if (loading) { return ( <div> // Shows a pulsing placeholder box with "Loading dataset…" // The user sees this for ~0.5 seconds while data is fetched </div> ); }

While data is loading, the component returns early with just a spinner. This prevents the rest of the page from rendering with empty/null data, which would cause errors.

5 · Data Computations – Preparing Charts & Statistics

After the dataset loads, a series of compute blocks prepare the data into formats that the charts can display. These run every time stats or data changes (i.e., when a new CSV is imported).

VariableWhat it computesUsed in
cityChurnData Per-city Active vs Churned counts, sorted by total size, top 8 cities City breakdown chart in card modal
ageChurnData Active/Churned/Rate per age group (e.g., "18-24", "25-34"…) Age group churn chart
spendBinData Groups customer spend into 5 bins: 0-25K, 25K-50K, 50K-75K, 75K-100K, 100K+ Spend distribution chart
ratingBinData Groups ratings into stars 1★ to 5★, counts Active/Churned for each Rating distribution chart
freqBinData Groups order frequency into 5 bins: 1-5, 6-10, 11-20, 21-30, 30+ Order frequency chart
loyaltyBinData Groups loyalty points into 5 ranges; directly filters data array Loyalty points chart
totalPieData Simple two-slice array: [{Active: N}, {Churned: N}] Total Customers pie chart
⚠️ Important design note: These calculations happen inside the component function, not in a useEffect. This means they re-compute on every render. For large datasets this is acceptable because React is efficient — but for even larger data, you would move these into useMemo() hooks.

📌 Example: How spendBinData is built

const spendBins = ['0-25K', '25K-50K', '50K-75K', '75K-100K', '100K+']; const spendBinData = spendBins.map((label, i) => { const ranges = [[0,25000],[25000,50000],[50000,75000],[75000,100000],[100000,Infinity]]; const [lo, hi] = ranges[i]; return { range: label, Active: stats.spendByChurn.active.filter(v => v >= lo && v < hi).length, Churned: stats.spendByChurn.inactive.filter(v => v >= lo && v < hi).length, }; });

For each spend range, it counts how many active customers and how many churned customers fall in that range. Recharts then renders this as a grouped bar chart automatically.

6 · CSV Import Feature – Upload Your Own Dataset

In the Dataset tab, users can upload their own .csv file to replace the default dataset. The importCSV function handles this entire process.

const importCSV = (e: React.ChangeEvent<HTMLInputElement>) => { const file = e.target.files?.[0]; // Get the selected file if (!file) return; setImporting(true); // Show "Importing…" on button const reader = new FileReader(); reader.onload = (ev) => { const text = ev.target?.result; // Raw CSV string Papa.parse(text, { header: true, // First row = column names skipEmptyLines: true, dynamicTyping: true, // Auto-detect numbers complete: (results) => { const cleaned = preprocessRecords(results.data); // Clean & validate setData(cleaned); // Update dashboard charts setStats(computeStats(cleaned)); // Recalculate all statistics setImportedData(cleaned); // Show in data table setImportStatus({ ok: true, msg: `✓ Loaded ${cleaned.length} records` }); }, }); }; reader.readAsText(file); // Start reading the file };

8 · AI Insights – Smart Pattern Detection

The AI Insights tab doesn't call an external AI API — instead, it applies statistical rules to the dataset to discover actionable patterns. Four computed values power these insights:

VariableFormula (simplified)Insight Generated
lowRatingChurnRate (Churned customers with rating < 2) ÷ (All customers with rating < 2) × 100 "Customers with <2 rating have X% churn probability"
delayedChurnRate Avg churn rate for late deliveries ÷ Avg churn rate for on-time deliveries "Delayed deliveries lead to X× higher churn rate"
highSpendRetention (Active customers spending ₹75K+) ÷ (All customers spending ₹75K+) × 100 "High spenders (₹75K+) have X% retention rate"
topChurnCity Sort cities by inactive count descending, take first "[City] is the city with highest number of churned customers"
📣 Key point for your guide: These insights are dynamic — they update automatically if a different CSV is uploaded. This simulates data-driven AI reporting. The insights are derived by applying statistical aggregation on the real dataset, not hardcoded strings.

3

Predict Churn Tab

AI-powered real-time churn risk prediction for a single customer

{activeTab === 'predict' && ( <PredictionForm /> )}

How PredictionForm works:

  1. User fills in customer attributes: rating, order frequency, spend, loyalty points, delivery status, payment method
  2. On submit, calls predictChurn() from dataset.ts — a rule-based scoring algorithm
  3. The algorithm assigns risk points based on thresholds (e.g., rating < 2.5 = high risk points)
  4. Outputs a churn probability % and a risk level: HIGH / MEDIUM / LOW
  5. A circular gauge (SVG-based) fills up to show the risk %
  6. Recommendations are shown explaining why the customer is at risk
  7. Prediction results are saved to Supabase database for future reference
The prediction logic is a deterministic ML-like algorithm — not a neural network, but a hand-crafted weighted scoring model based on domain knowledge of churn factors. This makes it interpretable and fast.

13 · Quick-Reference for Presentation

🎯 How to explain this project to your guide in 2 minutes

Opening line: "Our project FoodRetainAI is a web application that helps food delivery businesses identify and prevent customer churn using data analytics and AI predictions."

If asked about…Say this
What is DashboardPage.tsx? It is the main page of the application. It is a single React component that loads a customer dataset, computes statistics, and renders an interactive 6-tab analytics dashboard with charts, predictions, and reports.
What tech stack is used? React 18 with TypeScript (TSX), Vite build tool, Tailwind CSS for styling, Recharts for charts, Framer Motion for animations, PapaParse for CSV parsing, Supabase as the backend database.
How does data flow work? On page load, useEffect calls loadDataset() which fetches a CSV file. PapaParse converts it to an array of objects. computeStats() calculates KPIs. These are stored in React state. React re-renders the charts whenever state changes.
What is churn prediction? A user fills in customer attributes (rating, delivery status, spend, etc.) in the Predict Churn form. A rule-based algorithm scores these inputs using weighted thresholds to produce a churn probability % and risk level (LOW/MEDIUM/HIGH).
How are charts built? Using the Recharts library. Data is prepared into arrays of objects (e.g., [{range: "1-5★", Active: 200, Churned: 80}]). These arrays are passed to Recharts components like <BarChart>, <PieChart>, <LineChart> which render SVG charts automatically.
How does CSV import work? The user clicks "Import CSV". A hidden file input opens. When a file is selected, the browser FileReader reads it. PapaParse parses the CSV. The data is validated and cleaned by preprocessRecords(). The entire dashboard updates automatically with the new data.
What are the AI Insights? Statistical patterns computed from the dataset — for example, the churn rate of customers with rating < 2, the ratio of churn from delayed vs on-time deliveries, retention rate of high spenders. These are data-driven insights, not hardcoded values.
How is conditional rendering done? Using JavaScript's && operator inside JSX. For example: {activeTab === 'overview' && (<OverviewContent />)}. Only the matching tab renders; others do not exist in the DOM, keeping performance efficient.
How are animations added? Framer Motion's <motion.div> component wraps each tab content. initial={{opacity:0, y:10}} sets the starting state (invisible, shifted down). animate={{opacity:1, y:0}} is the target state. Framer Motion interpolates between them automatically.
How does authentication work? The signOut() function from localAuth.ts clears the user session. The Logout button calls signOut() then navigates to the home page using useNavigate(). Protected routes check for session tokens before rendering the dashboard.
